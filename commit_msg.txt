fix(SharedKernel): align all 16 Subscription event DTOs with Java JSON contract

Every C# event record now carries [property: JsonPropertyName("...")] attributes
matching Java's camelCase payload (e.g., sellerId, frozenUntil, bundleName).
Structural mismatches corrected:
  - BundleActivatedEvent: SubscriptionId -> Id (Java uses "id")
  - ChargebackProcessedEvent: replaced AmountCents with LockedCents/ChargebackCents
  - RefundIssuedEvent: removed RefundId/Reason, added ParticipantId
  - PlatformActivatedEvent: added UserId as nullable (Java POJO lacks it)
  - PlanChangedEvent: UserId/NewPlan made nullable (Java payload varies)
  - SellerClosedEvent: added From field (Java sometimes sends "from")

fix(post): use RabbitMqHealthCheck.CheckHealth() instead of GetProbeResult()
GetProbeResult() never throws for a disconnected bus; CheckHealth() returns
actual BusHealthStatus.
